04 — The Pronghorn Question

Your Internal Build Still Needs a Foundation

Pronghorn is smart architecture. It abstracts identity for downstream consumers so no single vendor owns the pipe. But the identity resolution underneath it is what determines whether every downstream decision is built on clean data or inherited fragmentation.

How Pronghorn Works Today

AmpID + Acxiom + Neustar → GMID

Pronghorn merges identity signals from multiple providers into a single GMID, then pipes it through Adobe AEP to Dentsu/M1 for activation. Marketing sees GMID. They don't see who resolved it.

Amperity's probabilistic matching is the engine inside Pronghorn. The 179M+ stitched profiles that feed GMID come from Amperity's identity resolution. Acxiom and Neustar provide supplemental signals. Amperity provides the graph.

What Changes with AI Agents

Static Graph → Living Intelligence

Today, the identity graph is a snapshot. It was built and it sits. New data arrives daily but the graph doesn't adapt. Match rules don't evolve. Accuracy doesn't improve.

The Identity Resolution Agent makes Pronghorn's foundation continuously self-improving. Every new data point refines the graph. Every match decision is benchmarked and adjusted. The foundation gets better, and everything downstream benefits automatically.
