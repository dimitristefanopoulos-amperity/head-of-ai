The why behind every merge

Every identity decision is traceable. We expose confidence scores, merge narratives, and variant detection so stakeholders can audit and trust the output.

Capability

Confidence Scoring

Every merge is scored with a numeric confidence level derived from field-level matching signals across name, email, phone, and address.

Capability

Merge Narrative

Human-readable explanation of why two records were merged, including which fields matched, which were fuzzy, and what the deciding signal was.

Capability

Name Variant Detection

Identifies and groups name variations (Pat/Patrick, Liz/Elizabeth) that Stitch resolved into the same unified customer.

Capability

Drift Monitoring

Tracks identity stability over time. Alerts when merge patterns shift unexpectedly, ensuring durable IDs remain durable.

09

Amperity's output is extensible

This proof of concept was built entirely on Amperity's public API surface. Everything here is replicable for any tenant.

API-First Architecture

Every data point on this page comes from Amperity's OAuth2-authenticated query API. The platform's output is not locked inside the UI — it is fully programmable.

Zero Infrastructure Overhead

This application runs as a single lightweight Flask server. No data warehouse, no ETL pipeline, no additional infrastructure. It queries Amperity in real time.

Client-Presentable by Default

The same data that powers internal operations can be surfaced in a format suitable for executive review, client presentations, and renewal conversations.

Replicable Across Accounts

This engine is not NFL-specific infrastructure. The same pattern can be deployed for any Amperity tenant with API access — any retail brand, any industry vertical.
